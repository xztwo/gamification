# Лабораторная работа 4  
## Интеграция и тестирование подсистемы геймификации сотрудников отеля

### Цель работы
Осуществить интеграцию подсистемы геймификации в информационную систему обучения сотрудников отеля, выполнить функциональное и интеграционное тестирование, а также организовать CI/CD-пайплайн для автоматической проверки качества.

### Задание
Выполнить требования из `Lab_4_2026.pdf`:  
- описать этапы CI/CD и задачи каждого этапа;  
- провести функциональное тестирование подсистемы;  
- провести интеграционное тестирование системы;  
- описать инструменты автоматизации тестирования;  
- сделать выводы по результатам.

### Краткое описание выполненных этапов
1. Проанализирована текущая реализация подсистемы геймификации в проекте React/Vite.  
2. Добавлена инфраструктура автотестов на Vitest и Testing Library.  
3. Реализованы функциональные тесты бизнес-логики геймификации.  
4. Реализованы интеграционные тесты UI-компонента лидерборда.  
5. Настроен CI в GitHub Actions с автоматическим запуском тестов и сборки.

### Этапы пайплайна CI/CD и задачи
1. **Checkout исходного кода**  
   Получение актуальной версии проекта из репозитория.
2. **Setup окружения Node.js**  
   Подготовка окружения для воспроизводимой сборки и тестов.
3. **Установка зависимостей (`npm ci`)**  
   Гарантированная установка версий из lock-файла.
4. **Запуск тестов (`npm run test:ci`)**  
   Проверка корректности логики геймификации и интеграции UI.
5. **Сборка (`npm run build`)**  
   Проверка, что приложение успешно собирается после изменений.

### План функционального тестирования и результаты
Функциональные тесты (`src/app/utils/gamification.test.ts`) покрывают:
- корректное определение уровня сотрудника по порогам баллов;
- расчет прогресса до следующего уровня;
- расчет вознаграждения с бонусами (качество, скорость, серия, ритм);
- обнаружение новых достижений;
- сохранение/загрузка профиля сотрудника из localStorage и миграция старых данных;
- ведение лидерборда (сортировка, ограничение топ-10, обновление существующей записи).

**Результат:** все тесты проходят успешно.

### План интеграционного тестирования и результаты
Интеграционные тесты (`src/app/components/Leaderboard.test.tsx`) проверяют:
- пустое состояние компонента при отсутствии данных;
- корректное отображение записей из localStorage;
- подсветку текущего пользователя;
- отображение времени прохождения в формате `mm:ss`.

**Результат:** интеграционные проверки проходят успешно.

### Инструменты автоматизации тестирования
- **Vitest** — быстрый тестовый раннер для Vite-проектов;
- **@testing-library/react** — тестирование пользовательского поведения компонентов;
- **@testing-library/jest-dom** — удобные матчеры для DOM-проверок;
- **GitHub Actions** — автоматический запуск тестов и сборки в CI.

### Где реализовано в проекте
- CI/CD: `.github/workflows/ci.yml`
- Скрипты тестирования: `package.json`
- Конфигурация тестов: `vite.config.ts`, `src/test/setup.ts`
- Функциональные тесты: `src/app/utils/gamification.test.ts`
- Интеграционные тесты: `src/app/components/Leaderboard.test.tsx`
